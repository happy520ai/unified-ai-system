# Phase1131 Local Trial Plan

## Objective

Check whether a local human can open the Future Minimal OS UI and quickly understand:

- What the screen is for.
- What to type.
- What the primary button does.
- What will not happen during preview.
- Which mode to choose next.

## Trial Steps

1. Open the local Workbench UI.
2. Observe the first screen without clicking anything.
3. Write down the first action the user thinks is safe.
4. Type one realistic task into the central task composer.
5. Click `预览执行方案`.
6. Read the recommendation and safety boundary.
7. Click `查看详情`.
8. Decide whether the next step is clear.
9. Record confusion, hesitation, wording problems, or layout issues.

## Success Criteria

- User understands that preview is safe.
- User does not think a real model call will happen.
- User does not think deployment will happen.
- User can describe Normal / God / Tianshu in plain language.
- User can identify one next step after preview.

## Evidence Rule

Only real user-provided notes or screenshots may be recorded as human trial evidence. Codex self-checks are automation evidence only.
