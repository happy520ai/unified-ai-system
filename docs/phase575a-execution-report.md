# Phase575A Execution Report

## Summary

Phase575A adds a real Chromium acceptance verifier for the Mission Control
sample dry-run first-use path. The verifier opens the required local URL,
clicks the sample dry-run entry, clicks the detail drawer action, checks the
structured dry-run evidence, and writes screenshots plus JSON evidence.

This phase is Codex browser product acceptance only. It does not collect real
non-Codex human feedback.

## Modified Files

- `apps/ai-gateway-service/src/ui/consolePage.js`
  - Added the missing in-page `workbenchApiClient.listApprovals()` bridge
    method as a read-only GET `/approvals` compatibility path. This prevents
    the real browser initialization path from recording
    `workbenchApiClient.listApprovals is not a function` as the last error.
- `tools/phase575a/validate-codex-real-browser-product-acceptance.mjs`
  - Starts or connects to the real local service on `127.0.0.1:3100`.
  - Opens Chromium through CDP.
  - Performs real clicks for `Start sample dry-run`, mode explanation,
    Security Shield, Evidence Replay, and `查看执行详情`.
  - Captures screenshots, rendered DOM, browser logs, and structured evidence.
- `docs/phase575a-codex-real-browser-product-acceptance.md`
  - Documents acceptance scope, URL, browser path, safety boundary, evidence,
    and seal rule.
- `docs/phase575a-execution-report.md`

## Current Result

Phase575A is currently blocked, not sealed.

The verifier detected that `127.0.0.1:3100` is already occupied by an existing
local service whose `/ui` HTML does not include the current
`workbenchApiClient.listApprovals()` bridge compatibility method. Using that
stale surface would reproduce the old initialization error and would not be a
valid acceptance run of the current code.

Observed blocker:

`stale_existing_service_on_3100`

The managed stop command returned no-op because the process is not managed by
the phase9c state file. A direct stop request for the process occupying 3100
was not approved in this environment, so this phase remains blocked rather
than bypassing the safety boundary.

After the stale 3100 service is stopped or refreshed by the user, rerun:

```powershell
node tools/phase575a/validate-codex-real-browser-product-acceptance.mjs
```
