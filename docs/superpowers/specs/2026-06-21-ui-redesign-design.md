# UI 重构最终方案 v3：意图优先 · 异步总控台（Ambient AI Console）

> **日期**：2026-06-21（v3 全量重写，吸收 5 项激进提升 + 安全网）
> **范围**：`unified-ai-system` 全部前端 UI + 多入口（网页/桌面/IM/通知/邮件）
> **状态**：v3 设计已确认方向，待 spec 审阅 → 实施计划
> **遵循契约**：`docs/design/codex-design-knowledge/PME_DESIGN.md`、`DO_NOT_DESIGN_LIKE_THIS.md`、`COMPONENT_PATTERNS.md`
> **后端支撑度**：经审计，本方案 70-80% 有真实后端支撑（见 §1.5 审计表），非画饼。

---

## 0. 一句话总结

> 老板的默认视图**几乎不存在**——一行字、一个呼吸灯、一个输入框。正常时什么都不长出来，异常才长出东西。功能不再住在页面里，而是**用户说一句话就被召唤出来**。任务说完就**关掉浏览器去做别的**，做完系统叫你，回来随时**回放**。网页只是五个入口之一（桌面小组件/IM 通知/语音/邮件/网页）。

从"更好的仪表盘"（v1）→"agent 任务流"（v2）→**"不像仪表盘的 AI 总控台"**（v3）。

## 1. 问题诊断与后端审计

### 1.1 根本矛盾（沿用 v1/v2）
老板侧（PME 极简）vs 运营侧（作战指挥室）两套相反哲学共用一个壳 → 按钮墙反复出现。

### 1.2 v2 的自我批评（v3 推翻它）
v2 用"任务流 + 深度旋钮"是进步，但仍是**装了任务流皮肤的仪表盘**：
- 默认主区仍是"一堆东西"，而老板的核心动作是**扫一眼确认没事**——正常时 UI 该几乎不存在。
- 仍假设"同步点击 + 等结果"，但 agent 系统本质异步（任务跑几分钟到几小时），绑住用户是反人类。
- 仍是单网页入口，但老板 90% 日常不该打开浏览器。
- 仍是预设布局，而最前沿是**任务驱动即时生成 UI**。

### 1.3 当前代码症状（沿用 v1/v2）
20 项扁平侧边栏（按钮墙）、80 个孤立 SSR Panel、三套色系漂移、品牌名混用、首页有按钮无输入区（phase1881）。

### 1.4 文档警告的失败模式（沿用）
不做按钮墙/工程后台首页/暴露 Phase；不把设计稿说成真实可用；UI 须无人解释可用。

### 1.5 ⭐ 后端能力审计（v3 可落地的根据）

经审计，本方案核心能力 **70-80% 有真实后端支撑**：

| 能力 | 状态 | 路径 | v3 用途 |
|---|---|---|---|
| NL→意图识别 | ✅ 存在（regex 级）| `owner-automation/chatActionIntentParser.js` | 输入框召唤功能 |
| 异步任务队列 | ✅ 存在（默认关闭）| `workforce/taskQueueManager.js`（`.data/workforce/` 持久化）| fire-and-forget |
| 执行 fabric | ✅ 存在 | `packages/workforce-execution-fabric` | 任务执行 |
| 推送通道 | ✅ 存在（dry-run 默认）| `im-connector-feishu`、`im-connector-wecom` | 日报/完成通知 |
| 证据回放 | ✅ 存在 | `workforceEvidenceTimeline.js`、evidence/ | 任务回放 |
| WebSocket 实时流 | ✅ 存在 | `http/webSocketServer.js`（`/ws`）| 实时进度推送 |
| 动作注册表 | ✅ 存在（仅 1 动作）| `registered-owner-actions.json` | 输入框可召唤的动作 |
| LLM 意图路由 | ⚠️ 需建（现 regex）| — | 提升意图命中率 |
| 浏览器推送 | ⚠️ 需建 | — | 桌面通知 |
| 执行开关开启 | ⚠️ 需授权 | 现 `executionEnabled=false` | 真实异步执行 |

**结论**：基础设施（队列/runner/证据/WS/连接器/注册表）齐全且已封板，**最后一公里**（LLM 意图路由、注册真实动作、安全开启执行、浏览器推送）需在建好的基础上补。v3 不是画饼。

## 2. 设计哲学（v3 核心）

### 2.1 第一性原理：形态服从核心动作

老板的核心动作 = **每天扫一眼，确认没事，然后去做别的**。
→ UI 最该像**一份晨报、一张便签、一个呼吸灯**，而不是 cockpit/Grafana/仪表盘。
→ **当一切正常时，UI 几乎不存在。** 只在异常时才长出更多东西。

### 2.2 五条激进提升（均已确认采用）

| 提升 | 内容 | 激进度 | 落地 |
|---|---|---|---|
| 1. 形态服从动作 | 默认几乎空白（一行字+呼吸灯+输入框），异常才长出 | 高 | P0 |
| 2. 杀死 IA | 输入框为主入口，IA 降为 fallback；常驻"更多"入口作安全网 | 高 | P0 |
| 3. 异步 fire-and-forget | 说一句话→"收到，我去做，做完叫你"→关浏览器→推送→回放 | 高 | P1 |
| 4. 多入口单中枢 | 网页只是五入口之一（桌面组件/IM/语音/邮件/网页）| 中 | P2 |
| 5. 生成式 UI | 布局任务驱动即时生成（远期北极星，P0-P3 不做）| 研究级 | 附录 |

### 2.3 安全网原则（激进但绝不崩）

每条激进设计都配降级机制，意图识别撑不住时不崩：
- **输入框识别失败** → 不卡死，降级为：模糊匹配建议 + "更多"入口（§3.2）。
- **异步执行未授权**（`executionEnabled=false`）→ 降级为 dry-run 预览 + "需要授权才能真跑"提示。
- **实时流断开** → 降级为轮询 + 断点续传。
- **生成式 UI 不稳** → P0-P3 用预设原子组件，生成式只做远期愿景。

## 3. 核心架构：Ambient Console

### 3.1 网页入口的默认形态（异常为零时）

```
╔═══════════════════════════════════════════════════════════╗
║                                                            ║
║                                                            ║
║                                                            ║
║              太极北斗 · 今天一切正常 ●                      ║
║              （呼吸灯 · 绿色 · 仅此一行）                    ║
║                                                            ║
║                                                            ║
║        ┌──────────────────────────────────────┐            ║
║        │  你想让系统做什么？                      │  ← 唯一入口 ║
║        └──────────────────────────────────────┘            ║
║                                                            ║
║      [帮我看今天日报]  [审批那 3 项]  [员工在干嘛]            ║  ← 3 个上下文 chip
║                                                            ║
║                                                            ║
║                                                    ⋯ 更多  ║  ← 安全网入口
╚═══════════════════════════════════════════════════════════╝
```

**关键**：
- 正常时**只有呼吸灯 + 输入框 + 3 chip + "更多"**。没有侧边栏、没有卡片网格、没有指标。
- 呼吸灯颜色 = 系统状态（绿=正常 / 黄=有待审批 / 红=有异常）。
- 3 个 chip 是**上下文相关的动态建议**（根据当前待办/异常生成），不是静态菜单。
- "更多"是常驻安全网（§3.2），点开是完整功能，**不必等识别失败**。

### 3.2 安全网："更多"入口（常驻，不依赖识别失败）

点"更多"→ 抽屉滑出完整功能区（即 v2 的 IA，但**降级为后备**）：

```
▾ 更多
  ▌ 对话（三模）
  ▌ 劳动力画布（空间/2D 可切换）
  ▌ 路由中心（30+ 模型 · 路由策略 · 成本）
  ▌ 审批中心（队列 · 历史）
  ▌ 知识库（RAG 检索）
  ▌ 安全（GVC 权限 · 审计 · 连接器）
  ▌ 设置（Provider · 密钥 · 备份）
  ▾ 高级模式（Phase · Evidence · Trace · Codex 上下文）
```

- 用户**随时主动**点开，不等识别失败。纯度与可达性平衡。
- 这保留了 v2 IA 的全部功能覆盖，只是从"默认平铺"变成"按需召唤"。

### 3.3 异常时 UI 才长出东西

呼吸灯变黄/红时，默认视图长出**最小必要的异常信息**：

```
╔═══════════════════════════════════════════════════════════╗
║              太极北斗 · ⚠ 有 1 件事要你处理 ●(黄)            ║
║                                                            ║
║   ┌──────────────────────────────────────────┐            ║
║   │ 待审批: "调用 DeepSeek 生成周报"  风险:低    │            ║
║   │                          [批准]  [拒绝]     │            ║
║   └──────────────────────────────────────────┘            ║
║                                                            ║
║        ┌──────────────────────────────────────┐            ║
║        │  你想让系统做什么？                      │            ║
║        └──────────────────────────────────────┘            ║
║                                                    ⋯ 更多  ║
╚═══════════════════════════════════════════════════════════╝
```

**核心**：信息按需出现，正常时 UI 几乎不存在。消灭"工程后台当首页"。

## 4. 交互范式：意图 → 确认 → fire-and-forget → 回放

### 4.1 一次完整交互的生命周期

```
1. 老板输入: "帮我把昨天客户反馈归类，找前三类问题"
2. 系统识别意图 + 风险（chatActionIntentParser + LLM 路由）
3. 系统回: "✓ 收到。归类客户反馈，预计 5 分钟。我去做，做完叫你。"
   - 若高风险: 先弹审批预览，批准后才跑
   - 若执行未授权(executionEnabled=false): 明确说"目前只能预览，要真跑需授权"
4. 老板关掉浏览器，去做别的 ← 真 fire-and-forget
5. 任务在 workforce 队列异步执行（taskQueueManager 持久化，不依赖浏览器）
6. 完成 → 推送（IM webhook / 桌面通知 / 邮件）
7. 老板回来 → 任务流从断点回放（workforceEvidenceTimeline）
```

### 4.2 输入框意图识别的降级链（安全网）

```
用户输入
   │
   ▼
① LLM 意图路由（命中?）
   │否
   ▼
② regex 意图解析（chatActionIntentParser，命中?）
   │否
   ▼
③ 模糊匹配已注册动作（fuzzy match registered-owner-actions）
   │否
   ▼
④ 降级: 显示"我没完全理解，你是想…" + 3 个最接近的建议
        + "更多"入口（用户总能主动找功能）
```

**绝不卡死**——四级降级，最后兜底是"更多"抽屉。

### 4.3 任务回放器（回来随时看）

回来后看到的不是"空白输入框"，而是**收件箱 + 回放器**：
- 进行中的任务：实时流（WebSocket `/ws`）+ 可暂停/介入
- 已完成任务：从证据时间线回放（像录像）
- 失败任务：`FailureFriendlyMessage`（失败原因 + 日志 + 一个下一步）

```
┌─ 你的任务 ──────────────────────────────────────────┐
│ ▎ 进行中 (1)                                        │
│   客户反馈归类  ─Tianshu─→ God 多候选  ▍生成中…       │
│   [查看证据] [介入] [暂停]                            │
│                                                      │
│ ▎ 已完成 (3)  · 今天                                 │
│   ☑ 晨报生成      ─Normal─→  8:00 已推送              │
│   ☑ 周报审批      ─God────→  你已批准                  │
│   ☑ 模型冒烟      ─Tianshu─→  全部通过                │
│                                                      │
│ ▎ 等你 (1)                                          │
│   ◌ 待审批: 调用 DeepSeek  [批准] [拒绝]              │
└──────────────────────────────────────────────────────┘
```

三模作为**模式带**嵌在每条任务里（Normal/God/Tianshu），不是独立面板。

## 5. 多入口单中枢（网页只是五分之一）

| 入口 | 用途 | 重量 | 后端支撑 |
|---|---|---|---|
| 桌面小组件 | 常驻呼吸灯 + 一行字 + 0 成本扫一眼 | 最轻 | 需建（呼应 zero-learning 启动器）|
| IM 通知（企微/微信）| 异步接收结果 + 日报推送 | 轻 | ✅ im-connector（dry-run 默认）|
| 语音 | "嘿太极，今天有问题吗" | 轻 | 需建（Web Speech API）|
| 邮件 | 每日 8:00 日报归档 | 轻 | 需建 |
| **网页** | 深操作/介入/审批/回放 | 最重 | ✅ 现有 |

**老板日常 90% 走前四个轻入口**，只有真正要介入/审批/排障才打开网页。网页从"唯一入口"降为"最重入口"。

## 6. 视觉系统（沿用 v2 收敛，加 ambient 极简档）

### 6.1 三档密度（v2 的 density 升级为三档）

| Token | ambient（默认/异常零）| calm（精简操作）| dense（深挖/运营）|
|---|---|---|---|
| 背景 | `#0a0e1a`，**极致留白** | 同，适度留白 | 同，紧凑 |
| 主区 | 一行字 + 呼吸灯 + 输入框 | 任务回放器 + 三卡 | 任务流 + 全 trace |
| 强调色 | 单一青 `#00e5ff`（呼吸灯/CTA）| 同 | 同 + 状态色 |
| 字体 | 中文优先，禁远程字体 | 同 | 同 |
| 圆角 | 12px | 12px | 8px |

**密度切换**：默认 ambient；用户点输入框开始操作 → calm；点"更多"深挖 → dense。**深度旋钮仍保留**（顶部 `◐深度`），但 ambient 档是"零异常时的默认"。

### 6.2 呼吸灯语义（唯一常驻视觉）

```
● 绿 = 一切正常，无待办无异常
● 黄 = 有待审批 / 有进行中任务
● 红 = 有异常 / 有失败 / 有阻塞
● 灰 = 系统离线 / 连接断开
```

**信任带**（v2 的常驻信任带）在 ambient 档**融入呼吸灯 tooltip**（hover 才展开 `0真实Provider调用 · 0密钥暴露 · 47证据已封存`），保持默认视图纯净；calm/dense 档显式常驻。

## 7. 劳动力空间画布（沿用 v2，带 PME 红线约束）

⚠️ 与 PME "不做花哨 3D / 大屏 cockpit" 红线冲突，三条约束让它合法：
1. **可降维**——一键坍缩成 2D 组织图（ambient 档不显示画布）。
2. **零装饰**——每节点/连线/动画映射真实员工状态/消息/分支。
3. **默认静态、交互才活**。

通过输入框"看看员工在干嘛"召唤，或"更多"→ 劳动力画布进入。

## 11. 验收标准（防"设计稿说成真实可用"）

- **ambient 纯度守卫**：异常为零时，首屏可点击元素 ≤ 4（呼吸灯/输入框/3chip算1组/更多）。
- **意图降级守卫**：任意输入不卡死，四级降级链兜底到"更多"抽屉。
- **异步诚实守卫**：`executionEnabled=false` 时，FireAndForgetAck 必须明确说"目前只能预览"，不假装真跑。
- **回放守卫**：关闭浏览器后任务不丢（taskQueueManager 持久化验证）；回来能从断点回放。
- **按钮墙守卫**：任何深度首屏顶级可点击 ≤ 3（ambient）/ ≤ 8（dense）。
- **死按钮守卫**：每个 `<button>` 有 handler + ButtonFeedbackState（Phase316A）。
- **零学习守卫**：ambient 档首屏无 Phase/verifier/evidence/trace/Provider Gate 字样（phase1939p）。
- **空间画布红线守卫**：①可坍缩 2D ②零装饰 ③默认静态。
- **视觉漂移守卫**：全站单一 token 源。
- **诚实声明**：不把截图/自动化测试等同于 production-ready/老板已认可。真实反馈走 `docs/owner-daily-use/records/`，自动化不计人工反馈。
