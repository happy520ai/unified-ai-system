> **日期**：2026-06-21（v4 基于亲眼查看真实渲染截图 + Phase1181 规范重定视觉地基）
> **范围**：`unified-ai-system` 全部前端 UI + 多入口
> **状态**：v4 设计已确认方向，待 spec 审阅 → 实施计划
> **遵循契约**：`docs/design/codex-design-knowledge/PME_DESIGN.md`、`DO_NOT_DESIGN_LIKE_THIS.md`、`COMPONENT_PATTERNS.md`、**`phase1181-1200-future-minimal-os-visual-spec.md`（视觉权威）**
> **方法论转向**：v1-v3 在宏观范式层面打架；v4 基于**亲眼查看 `phase1181_1200/google-chrome-screenshots/` 真实截图**，定位到中观/微观的像素级根因。

## 0. 一句话总结

> 前三版在宏观层面打架（角色/深度/输入框/异步），但**亲眼看了真实截图后**发现：Phase1181 已经把侧边栏砍到 5 图标、加了输入框和三模卡片——而老板反馈**仍然是 `oldWorkbenchFeeling=true`、`firstScreenTooDense=true`**。这证明砍侧边栏+加输入框**不解决根本问题**。真差距在：①深色 navy 本身就让人觉得"工程后台"；②一屏塞了 4 个焦点，密度过高；③`God Mode`/`CredentialRef` 等术语外泄；④输入框长得像表单。v4 = 把 v3 的 Ambient 范式**落到全新浅色冷调视觉系统**上，并逐条修掉这些像素级问题。

## 1. 关键转向：v4 为什么和 v3 不同

### 1.1 v1-v3 的方法论缺陷（自我批评）

**我前三版没有看过任何一张真实渲染图。** 我读代码结构、设计文档、导航配置，凭文档**想象** UI。这导致：
- v1 优化"20 项侧边栏"——但 Phase1181 早已砍到 5 图标，问题早不在这。
- v2/v3 加"输入框为主入口"——但 Phase1181 早已加了任务输入，老板**仍不满意**。
- 前三版默认"深空蓝玻璃拟态"——但 Phase1181 规范白纸黑字写"**白/冷灰/浅蓝，避免重深色侧边栏和密集 HUD**"，实现却做了深色 navy。**我前三版一直在强化这个错误**，而不是修正它。

### 1.2 亲眼看到的事实（`phase1181_1200/google-chrome-screenshots/`）

**初始屏**：深色 navy + 5 图标侧边栏 + "Good morning, Commander." + 三模卡片 + 任务输入面板 + 底部三栏（推荐模式/执行中 65%/安全边界）。

**展开态**：底部炸开 5 个 tab（Security Boundary / Provider/CredentialRef / Evidence Replay / Local evidence / Diagnostics），`CredentialRef`、`sandboxed`、`multi-step reasoning`、`God Mode`/`Tianshu Mode` 直接外露给老板。

**Phase1181 老板截图反馈（封板时的真实记录）**：
```
oldWorkbenchFeeling = true       // 仍像工程后台
leftSidebarTooHeavy = true       // 侧边栏太重
firstScreenTooDense = true       // 首屏太密
textareaLooksLikeForm = true     // 输入框像表单
bottomDockNotClean = true        // 底部 dock 不干净
floatingNoiseDetected = true     // 漂浮噪音
```

### 1.3 结论：真正的差距在像素，不在范式

| v1-v3 盯着的（宏观）| 截图暴露的真差距（中观/微观）|
|---|---|
| 角色壳 vs 深度旋钮 | 一屏塞了 4 个焦点（问候/输入/三模/进度+安全），**密度过高** |
| 仪表盘 vs 任务流 | **深色 navy 本身**就让人觉得"工程后台" |
| 输入框为主入口 | 输入框**长得像表单**（textarea + 字数计数）|
| 异步 fire-and-forget | **术语外泄**（God Mode / CredentialRef / sandboxed 直接给老板）|

## 2. 设计哲学（v4，视觉地基重定）

### 2.1 第一性原理（沿用 v3，但落地到视觉）

老板核心动作 = 扫一眼确认没事 → UI 正常时**几乎不存在**。

### 2.2 ⭐ 视觉基调：切到浅色冷调（遵 Phase1181 规范，推翻 v1-v3 深色）

**Phase1181 视觉规范（已封板，权威）原文**：
> "Use **white, cold gray, and light blue**. Surfaces are **translucent white** with low-noise borders. Accent is blue to violet only on the primary CTA and selected mode. **Avoid heavy dark sidebars, heavy panels, or dense HUD effects.**"

当前实现违背了这条规范（做成深色 navy）。v4 **严格遵规范**，这是从视觉根因上消除"工程后台感"的解药。

### 2.3 浅色冷调 token 系统

| Token | ambient（默认/异常零）| calm（操作）| dense（深挖）|
|---|---|---|---|
| 背景 | `#fafbfc` 近白冷调 | 同 | `#f4f6f8` 略沉 |
| 表面 | `rgba(255,255,255,0.85)` 半透明白 + 低噪边框 | 同 | 实线分隔 `#e4e8ec` |
| 文字主 | `#1a2233` 深墨蓝 | 同 | 同 |
| 文字辅 | `#5a6678` 冷灰 | 同 | `#8a94a3` |
| 强调（仅 CTA/选中）| 蓝→紫 `#3b82f6 → #7c6aef` 渐变 | 同 | 同 |
| 状态色 | 成功 `#10b981` / 警告 `#f59e0b` / 危险 `#ef4444` | 同 | 同 |
| 阴影 | 软阴影（shell/command surface），**禁重深色 panel/HUD** | 同 | 略深 |
| 字体 | 中文优先：微软雅黑/苹方/Segoe UI，**禁远程字体** | 同 | 同 |
| 圆角 | 12px | 12px | 8px |

**密度切换**：默认 ambient；输入框聚焦 → calm；点"更多"深挖 → dense。深度旋钮（顶部 `◐深度`）仍保留作显式开关。

## 3. 四条像素级修正（v4 核心新增，逐条对应截图问题）

### 3.1 ⭐ 修正 1：视觉根因——切浅色冷调（见 §2.2-2.3）
对应 `oldWorkbenchFeeling=true`。深色 navy → 浅色冷白。这是最大的单项提升。

### 3.2 ⭐ 修正 2：密度根因——一屏只允许一个焦点
对应 `firstScreenTooDense=true` + `floatingNoiseDetected=true`。

**规则**：任意一屏**只有一个视觉焦点**，其余退出首屏。

当前截图一屏有 4 个焦点（问候 / 输入 / 三模 / 进度+安全）→ v4 拆成：
- **无任务时**：焦点 = 输入框。问候语融入背景（大而淡），三模隐藏（提交后系统自动选，只在"想手动选"时展开），进度/安全退出首屏（进呼吸灯 tooltip）。
- **有任务时**：焦点 = 当前执行流。输入框收缩为顶部一行，其余退出。
- **有异常时**：焦点 = 异常卡（呼吸灯变黄/红时长出）。其余退出。

**一屏永远只有一个"主角"**，配角全部淡入背景或移出首屏。

### 3.3 ⭐ 修正 3：术语根因——老板词汇表硬墙
对应 `God Mode`/`Tianshu Mode`/`CredentialRef`/`sandboxed`/`multi-step reasoning` 外露。

复用 `docs/design/codex-design-knowledge/CHINESE_UI_COPY_RULES.md` 翻译表，但**升级为 UI 层硬墙**：
- `God Mode` → "严谨"（多候选评审）
- `Tianshu Mode` → "智能"（系统自动选最优）
- `Normal Mode` → "极速"（直接出结果）
- `CredentialRef` → "（已配置，不显示）"
- `sandboxed` → "隔离运行"
- `evidence/trace/phase/verifier` → 全部进高级抽屉，老板视图**字面不存在**

**实现**：一个 `ownerCopyFilter()` 中间层，所有文案经过它才渲染到老板视图。违反词汇表的文案在老板深度下**渲染前被替换或隐藏**。

### 3.4 ⭐ 修正 4：表单感根因——输入框从 textarea 变对话气泡
对应 `textareaLooksLikeForm=true`（致命反馈：让老板觉得"我要填表"）。

**当前**：带字数计数、带边框、固定尺寸的 textarea = 表单感。
**v4**：
- 无边框、无字数计数、大留白、口语化占位符（"今天想让系统帮你做什么？"）
- 圆角对话气泡形态，回车即提交
- 聚焦时**柔和蓝光晕**（遵 Phase1181："Mission input receives a soft blue glow on focus"）
- 三模选择**提交后才出现**（默认系统自动选，想手动选时展开），不再首屏平铺三张卡片

> 让老板觉得"我在跟系统说话"，不是"我在填表"。

## 8. 验收标准（v4 新增像素级守卫）

沿用 v3 全部守卫，**新增**：
- **浅色基调守卫**：首屏背景明度 ≥ `#f4f6f8`；无深色 navy panel/sidebar（遵 Phase1181 规范）。当前深色 navy 实现视为**违反已封板规范**，必须改。
- **单焦点守卫**：任意状态下一屏只有一个视觉焦点；焦点外元素亮度/对比度降至背景级。
- **术语硬墙守卫**：老板视图（ambient/calm 深度）渲染文本经 `OwnerCopyFilter`，不含 `God/Tianshu/Normal Mode`/`CredentialRef`/`sandboxed`/`evidence`/`trace`/`phase`/`verifier` 原词（自动化测试扫描 DOM）。
- **表单感守卫**：主输入框无字数计数、无边框（聚焦蓝光晕除外）、口语化占位符。

（其余守卫沿用 v3 §11：意图降级/异步诚实/回放/按钮墙/死按钮/零学习/空间画布红线/视觉漂移/诚实声明。）

## 10. v4 相对前三版，到底解决了什么（诚实对照）

前三版的范式（角色/深度/输入框/异步）**全部保留**在 v4。v4 的增量贡献是：

1. **方法论升级**：从"凭文档想象"到"亲眼看真实截图"——定位到前三版全错过的像素级根因。
2. **视觉地基重定**：从深色 navy（违反已封板规范）→ 浅色冷调（遵 Phase1181 规范）。这是"工程后台感"的**视觉根因**，前三版全在强化它。
3. **四条像素级修正**：浅色 / 单焦点 / 术语硬墙 / 对话气泡——每条都**直接对应** Phase1181 老板截图反馈里的一个 `=true`。
